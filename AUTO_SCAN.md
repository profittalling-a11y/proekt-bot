# Автоматическое сканирование ликвидных пар

## 🔍 Как это работает

Вместо ручного указания криптопар, бот автоматически сканирует Bybit и выбирает самые ликвидные пары на основе объема торгов за 24 часа.

## ⚙️ Настройка

### Включить автосканирование

В файле `.env`:

```env
AUTO_SCAN=true
MIN_VOLUME_24H=1000000
MAX_SYMBOLS=10
```

**Параметры:**
- `AUTO_SCAN=true` - включить автоматическое сканирование
- `MIN_VOLUME_24H=1000000` - минимальный объем за 24ч (1 млн USDT)
- `MAX_SYMBOLS=10` - максимум пар для торговли

### Отключить автосканирование (ручной режим)

```env
AUTO_SCAN=false
SYMBOLS=BTCUSDT,ETHUSDT,SOLUSDT
```

## 📊 Что делает сканер

1. **Получает все пары** с Bybit (perpetual futures)
2. **Фильтрует по объему** - только пары с объемом ≥ 1M USDT за 24ч
3. **Сортирует по ликвидности** - самые ликвидные первыми
4. **Берет топ N** - например, топ 10 самых ликвидных
5. **Определяет плечо** - автоматически для каждой пары (макс 50x)
6. **Генерирует конфиг** - обновляет SYMBOLS и LEVERAGE_CONFIG

## 🔄 Обновление списка

### Автоматическое обновление

Сканирование происходит:
- **При запуске бота** - всегда
- **Каждые 24 часа** - автоматически (опционально)

### Ручное обновление

Просто перезапустите бота - он пересканирует рынок.

## 📈 Преимущества автосканирования

### 1. Всегда актуальные пары
- Рынок меняется
- Новые монеты появляются
- Старые теряют ликвидность
- Бот автоматически адаптируется

### 2. Максимальная ликвидность
- Торгуете только ликвидными парами
- Минимальный slippage
- Быстрое исполнение ордеров

### 3. Диверсификация
- Автоматически распределяет по топ-10
- Снижает риск концентрации
- Больше возможностей для сигналов

### 4. Оптимальное плечо
- Автоматически определяет максимальное плечо для каждой пары
- Учитывает ограничения биржи
- Безопасно (макс 50x)

## ⚠️ Важные моменты

### Минимальный объем

**1 млн USDT** - это хороший баланс:
- ✅ Достаточная ликвидность
- ✅ Популярные монеты
- ✅ Обычно 30-50 пар

Можно изменить:
```env
MIN_VOLUME_24H=5000000  # 5M USDT - только топовые
MIN_VOLUME_24H=500000   # 500K USDT - больше пар
```

### Количество пар

**10 пар** - оптимально:
- ✅ Хорошая диверсификация
- ✅ Управляемое количество
- ✅ Не перегружает систему

Можно изменить:
```env
MAX_SYMBOLS=5   # Консервативно
MAX_SYMBOLS=20  # Агрессивно (больше нагрузка)
```

### Testnet vs Mainnet

**Важно:** На testnet может быть меньше пар!

Testnet обычно имеет:
- BTCUSDT
- ETHUSDT
- Несколько других топовых

Mainnet имеет 100+ пар.

## 🛠️ Ручной режим

Если хотите контролировать пары вручную:

```env
AUTO_SCAN=false
SYMBOLS=BTCUSDT,ETHUSDT,SOLUSDT
LEVERAGE_CONFIG=BTCUSDT:50,ETHUSDT:50,SOLUSDT:25
```

**Когда использовать:**
- Тестирование конкретных пар
- Специфическая стратегия
- Ограниченный список монет
