# 📸 ВИЗУАЛЬНАЯ ИНСТРУКЦИЯ: Настройка TradingView за 5 минут

## ✅ ШАГ 1: Запуск системы (2 минуты)

### Терминал 1 - Запуск вебхук-сервера:
```
1. Откройте командную строку (Win+R → cmd → Enter)
2. Выполните:
   cd "C:\Users\desin\OneDrive\Рабочий стол\Proekt Bot"
   start_webhook.bat
   
3. Вы увидите:
   ========================================
   TRADINGVIEW WEBHOOK SERVER STARTING
   ========================================
   Webhook URL: http://0.0.0.0:5001/tradingview/webhook
   📡 Waiting for TradingView webhooks...
```

### Терминал 2 - Запуск ngrok:
```
1. Откройте НОВОЕ окно командной строки
2. Выполните:
   ngrok http 5001
   
3. Вы увидите такую картину:
   
   ┌─────────────────────────────────────────────────────┐
   │ ngrok                                                │
   │                                                      │
   │ Session Status    online                            │
   │ Forwarding        https://abc123.ngrok.io -> 5001  │  ← ЭТО ВАШ URL!
   │                                                      │
   └─────────────────────────────────────────────────────┘
   
4. СКОПИРУЙТЕ URL из строки "Forwarding"
   Например: https://abc123.ngrok.io
```

**⚠️ ВАЖНО: Оставьте ОБА окна открытыми!**

## ✅ ШАГ 2: Настройка TradingView (3 минуты)

### 2.1 Откройте график

```
1. Зайдите на: https://www.tradingview.com/chart/
2. В поиске введите: BTCUSDT
3. Выберите пару и биржу (например, Binance BTCUSDT)
```

### 2.2 Добавьте индикатор Supertrend

```
1. Нажмите кнопку "Indicators" вверху
2. В поиске введите: Supertrend
3. Выберите "Supertrend" (by KivancOzbilgic или стандартный)
4. Настройте параметры:
   - ATR Length: 10
   - Factor: 3.0
```

Индикатор появится на графике:
- 🟢 **Зеленая линия** = восходящий тренд (BUY)
- 🔴 **Красная линия** = нисходящий тренд (SELL)

## ⚠️ ЧАСТЫЕ ОШИБКИ И РЕШЕНИЯ

### ❌ Ошибка: "Webhook не приходит"

**Решение:**
```
1. Проверьте что ngrok запущен и показывает "online"
2. Проверьте URL в TradingView:
   - Должен быть: https://abc123.ngrok.io/tradingview/webhook
   - Не должно быть пробелов или лишних символов
3. Проверьте что в алерте включено "Send Webhook"
```

### ❌ Ошибка: "Invalid symbol"

**Решение:**
```
В Message используйте формат: "symbol": "BTCUSDT"
БЕЗ пробелов, БЕЗ слешей, БЕЗ тире
Правильно: BTCUSDT
Неправильно: BTC/USDT, BTC-USDT, BTC USDT
```

### ❌ Ошибка: "Insufficient balance"

**Решение:**
```
1. Пополните testnet аккаунт OKX
2. Или уменьшите FIXED_POSITION_SIZE в .env:
   FIXED_POSITION_SIZE=0.5
```

### ❌ Ошибка: "Trading not allowed"

**Решение:**
```
Проверьте настройки риск-менеджмента в .env:
- MIN_BALANCE=10.0 (минимальный баланс)
- MAX_POSITIONS=10 (максимум позиций)
- MAX_DAILY_LOSS=20.0 (макс. дневной убыток %)
```
