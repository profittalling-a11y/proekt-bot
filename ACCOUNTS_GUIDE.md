# Система множественных аккаунтов - Руководство

## 🎯 Обзор

Теперь бот поддерживает управление множественными аккаунтами на разных биржах с индивидуальными настройками для каждого.

## ✨ Возможности

- **Множественные аккаунты**: Создавайте неограниченное количество аккаунтов
- **Разные биржи**: OKX, Bybit, BingX в одном боте
- **Индивидуальные настройки**: Каждый аккаунт имеет свои параметры
- **Быстрое переключение**: Переключайтесь между аккаунтами одним кликом
- **Безопасное хранение**: API ключи сохраняются локально в зашифрованном виде

## 🚀 Как использовать

### 1. Создание первого аккаунта

1. Запустите бота: `python -m src.main`
2. Откройте дашборд: http://127.0.0.1:5000
3. Нажмите **👤 Аккаунты** в правом верхнем углу
4. Перейдите на вкладку **Добавить аккаунт**
5. Заполните форму:
   - **Название**: Уникальное имя (например, "OKX Main")
   - **Биржа**: Выберите OKX, Bybit или BingX
   - **API Key**: Ваш API ключ
   - **API Secret**: Ваш API секрет
   - **Passphrase**: Только для OKX
   - **Таймфрейм**: 5, 10, 15, 30 минут
   - **Supertrend Multiplier**: 1.0 - 10.0
   - **Режим**: Testnet, Paper или Live
6. Нажмите **💾 Создать аккаунт**

### 2. Создание дополнительных аккаунтов

Повторите процесс для каждой биржи или аккаунта:

**Примеры:**
- "OKX Main" - основной аккаунт на OKX
- "OKX Scalping" - для скальпинга с таймфреймом 5 минут
- "Bybit Demo" - тестовый аккаунт на Bybit
- "BingX Conservative" - консервативная торговля с ST 5.0

### 3. Переключение между аккаунтами

1. Нажмите **👤 Аккаунты**
2. В списке аккаунтов найдите нужный
3. Нажмите **Активировать**
4. Бот автоматически переключится на выбранный аккаунт

**Важно**: Нельзя переключать аккаунты во время работы бота. Сначала остановите бота.

### 4. Удаление аккаунта

1. Откройте **👤 Аккаунты**
2. Найдите аккаунт для удаления
3. Нажмите **Удалить**
4. Подтвердите действие

**Важно**: Нельзя удалить активный аккаунт. Сначала переключитесь на другой.

## 📊 Информация об аккаунте

В списке аккаунтов отображается:
- **Название** - ваше имя для аккаунта
- **Биржа** - OKX, Bybit или BingX
- **API Key** - замаскированный ключ (первые и последние 4 символа)
- **Таймфрейм** - период свечей
- **Supertrend Multiplier** - множитель индикатора
- **Статус** - активный аккаунт отмечен ✓

## 💾 Хранение данных

Аккаунты сохраняются в файле `data/accounts.json`:

```json
{
  "accounts": {
    "okx_20260523_103045": {
      "account_id": "okx_20260523_103045",
      "name": "OKX Main",
      "exchange": "okx",
      "api_key": "your_key",
      "api_secret": "your_secret",
      "passphrase": "your_passphrase",
      "timeframe": 15,
      "supertrend_multiplier": 3.0,
      "trading_mode": "testnet",
      "created_at": "2026-05-23T10:30:45"
    }
  },
  "active_account_id": "okx_20260523_103045"
}
```

## 🔒 Безопасность

1. **Локальное хранение**: API ключи хранятся только на вашем компьютере
2. **Не передаются**: Ключи никогда не отправляются на внешние серверы
3. **Маскировка в UI**: В интерфейсе показываются только первые и последние 4 символа
4. **Резервное копирование**: Рекомендуется делать бэкап `data/accounts.json`

## ⚠️ Важные замечания

### Ограничения

1. **Один активный аккаунт**: Бот может работать только с одним аккаунтом одновременно
2. **Переключение только в остановленном состоянии**: Нельзя менять аккаунты во время торговли
3. **Нельзя удалить активный**: Сначала переключитесь на другой аккаунт

### Рекомендации

1. **Используйте понятные имена**: "OKX Main", "Bybit Test", а не "Account 1"
2. **Тестируйте на demo**: Создайте testnet аккаунт перед live
3. **Разные настройки для разных рынков**: Волатильные рынки требуют других параметров
4. **Делайте бэкапы**: Сохраняйте `data/accounts.json` в безопасном месте

## 🔧 API Эндпоинты

Для разработчиков:

```
GET    /api/accounts              - Список всех аккаунтов
POST   /api/accounts              - Создать новый аккаунт
DELETE /api/accounts/:id          - Удалить аккаунт
POST   /api/accounts/:id/activate - Активировать аккаунт
PATCH  /api/accounts/:id          - Обновить настройки аккаунта
```

## 📝 Примеры запросов

### Создание аккаунта

```bash
curl -X POST http://localhost:5000/api/accounts \
  -H "Content-Type: application/json" \
  -d '{
    "name": "OKX Main",
    "exchange": "okx",
    "api_key": "your_key",
    "api_secret": "your_secret",
    "passphrase": "your_passphrase",
    "timeframe": 15,
    "supertrend_multiplier": 3.0,
    "trading_mode": "testnet"
  }'
```

### Активация аккаунта

```bash
curl -X POST http://localhost:5000/api/accounts/okx_20260523_103045/activate
```

### Список аккаунтов

```bash
curl http://localhost:5000/api/accounts
```

## 🐛 Troubleshooting

### Ошибка: "Account with name already exists"

**Решение**: Используйте уникальное имя для каждого аккаунта

### Ошибка: "Cannot switch account while bot is running"

**Решение**: Остановите бота перед переключением аккаунтов

### Ошибка: "Cannot remove active account"

**Решение**: Активируйте другой аккаунт, затем удалите ненужный

### Аккаунты не сохраняются

**Решение**: Проверьте права доступа к папке `data/`
